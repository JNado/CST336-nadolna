The Domain Name System is a hierarchical system. This means that there are different levels, and each level has a function.

The easiest way to describe the DNS hierarchy is to describe the path of a query.

An user makes a query that contains a URL to the root DNS server, which is the highest level of DNS. The root server may or may not already know the IP address that that URL translates to, and if it doesn't know it forwards it to the level below it. That level is that of the authoritative name servers, each of which belongs to a specific Domain Name Space. The authoritative name servers are either going to know the translation or are going to forward it to the lower level domain name servers, which are the next lower level of the hierarchy. These servers are the lowest level we shall discuss here, and are practically guarenteed to know the translation.

Now that the translation has been found, the DNS query can return the IP address that corresponds to the queried URL to the user, and they can continue surfing the Web.
